Наши Роботы обожают игры, в которых надо быстро считать. Они постоянно ищут новые игры и вот сегодня новая - "Пронумерованные треугольники". Игроки берут 6 фишек из кучи. Каждая фишка имеет форму равностороннего треугольника с тремя числами на краях. Вы можете двигать, вращать и переворачивать фишки, чтобы сформировать шестиугольник. Шестиугольник считается правильным, если соприкасающиеся края треугольников имеют одинаковые числа.

Для правильного шестиугольника считаются очки - это сумма чисел на внешних шести сторонах. Цель игрока - найти комбинацию, которая принесет максимальное количество очков.

Каждая фишка представлена, как список трех положительных чисел. Вам необходимо найти максимально возможное количество очков для 6 выбранных фишек. Если из данных фишек нельзя образовать правильный (по правилам игры правильный) шестиугольник, то верните 0.

Входные данные: Информация о фишках, как список (list) списков, каждый с тремя целыми числами (int).

Выходные данные: Максимально возможное количество очков, как целое число (int).

Примеры: 

checkio([[1, 4, 20], [3, 1, 5], [50, 2, 3],
         [5, 2, 7], [7, 5, 20], [4, 7, 50]]) == 152
checkio([[1, 10, 2], [2, 20, 3], [3, 30, 4],
         [4, 40, 5], [5, 50, 6], [6, 60, 1]]) == 210
checkio([[1, 2, 3], [2, 1, 3], [4, 5, 6],
         [6, 5, 4], [5, 1, 2], [6, 4, 3]]) == 21
checkio([[5, 9, 5], [9, 6, 9], [6, 7, 6],
         [7, 8, 7], [8, 1, 8], [1, 2, 1]]) == 0

Зачем это нужно: Данная концепция используется в оптимизации задач. Есть ограничения, но вам нужно выжать максимум при этих ограничениях.

Предусловия:
len(chips) == 6
all(all(0 < x < 100 for x in ch) for ch in chips)
